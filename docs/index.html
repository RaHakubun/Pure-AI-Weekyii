The Engine

Contribution I: Week-Driven Auto-Progress State Machine

Past / Present / Pending are not views, they are enforced timeline states.

Step 01 One Present Week

The current week is the only active execution realm.

Step 02 Daily Auto Advance

Today shifts with real time, not with manual refreshing.

Step 03 Cross-Week Rollover

Past absorbs old week, Pending promotes new week.

Step 04 Habit Reinforcement

Recorded tasks carry clear weekly time pressure.

  • Week as the modeling core gives long-term companionship and supervision, not fragmented daily anxiety.
  • Any explicitly recorded task gains time perception immediately, making procrastination psychologically harder.
  • The state machine, not user memory, guarantees timeline progression integrity.

The Ritual

Contribution II: Commitment & Accountability Mechanism

Start is commitment · Expire is forgetting · Complete is reward · Neglect has penalty.

  • Every day has an explicit deadline (`kill_time`), and every task has a clear owner: you.
  • Draft zone is editable. After start, tasks enter Focus/Frozen and sequencing discipline begins.
  • A lightweight regret path exists: tasks can move later, but commitment pressure remains visible.
  • Expired tasks are forgotten forever, forcing respect for daily promises instead of backlog hoarding.
Start = Commitment Expire = Forgetting Complete = Reward Neglect = Penalty
